# 명명규칙

## 업무 명칭

### 개요

* 시스템명의 약자는 HM 이라고 가정한다.
* 업무명은 다음과 같이 나눈다
  * om 시스템 관련 (권한관리, 메뉴관리, 사용자관리 등)
  * 나머지는 개발자들과 협의하여 정한다.

## URL

### 개요 

* 조회, 등록, 상세, 수정 화면은 각각 주소를 다르게 구성한다. 왜냐하면 주소를 다르게 처리하여야 권한 처리가 가능하기 때문이다.
* URL 구성 시에는 다음과 같이 구성한다
  * /시스템명/업무명/화면번호/세칙

### 세칙

1. 조회
   * 화면
     * /hm/cs/3110
   * 검색
     * 기본 화면에서 검색 버튼을 눌렀을 경우
     * /hm/cs/3110/find
2. 등록
   * 화면
     * /hm/cs/3115
   * 등록 처리
     * /hm/cs/3115/regist
3. 상세
   * 화면
     * /hm/cs/3120
   * 삭제 처리
     * /hm/cs/3120/delete
4. 수정
   * 화면
     * /hm/cs/3125
   * 수정 처리
     * /hm/cs/3125/modify
5. 기타
   * 등록/상세/수정 외에는 의미에 맞게 URL을 구성하되 개발자들과 협의하여 정한다.

## 패키지

### 개요

* 소문자만 사용한다.
* 언더바( _ ) 문자를 포함 특수문자를 사용하지 않는다.
* 메인 패키지는 도메인명을 역순으로 하여 명명한다.
  * 네이버의 도메인명은 **www.naver.com** 이므로 패키지명은 **com.naver**가 된다.
* 도메인이 따로 정해지지 않은 경우에는 프로그램 이름을 기준으로 해 구성한다.
  * 회사의 프레임워크는 회사의 도메인 접미어인 **.com**과 회사의 약칭인 **ain** 그리고 프레임워크 이름인 **ainframework**를 조합하여 **com.ain.ainframework**로 명명하였다.

### 세칙

#### src/main/java

* com.ain.ainframework 메인 패키지 

1. cmmn 공통
   * annotation 어노테이션
   * config 기본 설정
   * datatype 자료형
   * exception 예외
   * interceptor 인터셉터
   * ibatis (영속성 프레임워크가 iBatis인 경우)
   * mybatis (영속성 프레임워크가 MyBatis 인 경우)
   * policy 정책 (파일규칙 정책 및 비밀번호 정책 등)
   * security 스프링 시큐리티 설정
   * util 유틸리티
     * crypto 암복호화
     * excel 엑셀
     * http 웹 (헤더 등)
     * view 뷰
2. controller 컨트롤러
   * om 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
   * cm 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
3. service 서비스 인터페이스
   * om 인터페이스용 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
   * cm 인터페이스용 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
   * impl 서비스 구현체
     * om 구현체용 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
     * cm 구현체용 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
4. dao (영속성 프레임워크가 iBatis인 경우)
   * om 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
   * cm 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
5. mapper (영속성 프레임워크가 MyBatis 인 경우)
   * om 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
   * cm 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
6. vo (파일, 페이징, 세션 등)

#### src/main/resources

1. META-INF
   * resources
     * ibatis
       * sql 매핑 파일 (SQL)
         * om 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
         * cm 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
     * mybatis
       * mappers 매핑 파일 (SQL)
         * om 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
         * cm 각 업무별 패키지 (예를 들어 기술하였으므로 업무에 맞게 세팅)
     * spring 스프링 설정 파일 (ehcache.xml, sql-map-config.xml, tiles.xml)
2. static 정적 소스
   * css 스타일시트
   * images 이미지
   * js 자바스크립트
3. application.yml 기본 설정에 사용되는 프로퍼티

## 클래스

### 개요

* 앞 글자는 대문자로 시작한다.
* 카멜 형식으로 작성한다.
  * MemberController
  * MemberService
  * MemberAuthenticationProvider
* 언더바( _ ) 문자를 포함 특수문자를 사용하지 않는다.

### 세칙

#### 컨트롤러

* 이름은 시스템명 + 업무명 + 화면번호 + Controller 로 한다.
  * 시스템 약칭이 HM이고 업무명이 시스템 관리의 약칭이 OM이며 화면번호가 0000인 경우
  * HmOm0000Controller

#### 서비스

* 인터페이스의 이름은 시스템명 + 업무명 + 화면번호 + Service 로 한다.
  * 시스템 약칭이 HM이고 업무명이 시스템 관리의 약칭이 OM이며 화면번호가 0000인 경우
  * HmOm0000Service
* 구현체의 이름은 시스템명 + 업무명 + 화면번호 + ServiceImpl 로 한다.
  * 시스템 약칭이 HM이고 업무명이 시스템 관리의 약칭이 OM이며 화면번호가 0000인 경우
  * HmOm0000ServiceImpl

#### 영속성 계층 (DAO 혹은 MAPPER)

* iBatis의 경우 이름은 시스템명 + 업무명 + 화면번호 + Dao 로 한다.
  * 시스템 약칭이 HM이고 업무명이 시스템 관리의 약칭이 OM이며 화면번호가 0000인 경우
  * HmOm0000Dao
* MyBatis의 경우 이름은 시스템명 + 업무명 + 화면번호 + Mapper 로 한다.
  * 시스템 약칭이 HM이고 업무명이 시스템 관리의 약칭이 OM이며 화면번호가 0000인 경우
  * HmOm0000Mapper

#### 기타

* 기능에 맞게 개발자들과 협의하여 정한다.

## 메소드

### 개요

- 앞 글자는 대문자로 시작한다.
- 카멜 형식으로 작성한다.
  * findMemberNo
  * registMember
  * eraseMember
- 언더바( _ ) 문자를 포함 특수문자를 사용하지 않는다.

### 세칙

#### 컨트롤러

* 메소드에 붙어 있는 @RequestMapping 어노테이션의 value 값에 따라 카멜 형식으로 지정한다.
* @RequestMapping이 클래스에 붙어 있으나 메소드에 @RequestMapping(value = "") 로 되어 있는 경우에는 메소드명을 main으로 정한다.

1. 조회
   * 화면
     * @RequestMapping(value = "") 이면 메소드명은 main
   * 검색
     * @RequestMapping(value = "/find") 이면 메소드명은 find
     * @RequestMapping(value = "/find/member/page/1") 이면 메소드명은 findPage (기본 조회 외에 추가로 조회가 필요한 경우)
2. 등록
   * 화면
     * @RequestMapping(value = "") 이면 메소드명은 main
   * 등록 처리
     * @RequestMapping(value = "/regist") 이면 메소드명은 regist
     * @RequestMapping(value = "/regist/id") 이면 메소드명은 registId (기본 등록 외에 추가로 등록이 필요한 경우)
3. 상세
   * 화면
     * @RequestMapping(value = "") 이면 메소드명은 main
   * 삭제 처리
     * @RequestMapping(value = "/delete") 이면 메소드명은 delete
     * @RequestMapping(value = "/delete/file") 이면 메소드명은 deleteFile (기본 삭제 외에 추가로 삭제가 필요한 경우)
4. 수정
   * 화면
     * @RequestMapping(value = "") 이면 메소드명은 main
   * 수정 처리
     * @RequestMapping(value = "/modify") 이면 메소드명은 modify
     * @RequestMapping(value = "/modify/item") 이면 메소드명은 modifyItem (기본 삭제 외에 추가로 수정이 필요한 경우)
5. 기타
   * 검사 및 유효성 체크 시 메소드명은 check + 기능명
   * 다운로드 시 메소드명은 download + 기능명
   * 그 외에는 개발자들과 협의하여 정한다.

#### 서비스

1. 조회
   * 여러 건 조회 (목록)
     * find + 기능명 + List
     * findMemberList 회원 목록을 조회
   * 단일 건 조회 (상세)
     * find + 기능명
     * findMember 단일 회원 조회
2. 등록
   * regist + 기능명
   * registMember 회원 데이터를 등록
3. 수정
   * modify + 기능명
   * modifyMember 회원 데이터를 수정
4. 기타
   * 검사 및 유효성 체크 시 메소드명은 check + 기능명
   * 업로드 시 메소드명은 upload + 기능명
   * 다운로드 시 메소드명은 download + 기능명
   * 그 외에는 개발자들과 협의하여 정한다.

#### 영속성 계층 (DAO 혹은 MAPPER)

1. 조회
   * 여러 건 조회 (목록)
     * select + 기능명 + List
     * selectMemberList 회원 목록을 조회
   * 단일 건 조회 (상세)
     * select + 기능명
     * selectMember 단일 회원 조회
2. 등록
   * insert + 기능명
   * insertMember 회원 데이터를 등록
3. 수정
   * update + 기능명
   * updateMember 회원 데이터를 수정
4. 병합
   * merge + 기능명
   * mergeMember 회원 데이터를 병합 (데이터가 중복이 아니면 등록하고 데이터가 중복이면 수정하는 기능)
5. 프로시저
   * callProcedure + 기능명
   * callProcedureMemberSync 회원 데이터를 동기화하는 프로시저를 호출
6. 기타
   * 그 외에는 개발자들과 협의하여 정한다.
